Environmental Station Measurements Display Application

An application for displaying and documenting values retrieved from a Technology Club of Thrace Environmental Station.

1. Application Description

How is this application used?

It is a display, visualization and documentation system for values retrieved from a specific station operated by the Technology Club of Thrace.

Its goal is to present the station's current measurements in a simple, understandable and reliable way on public televisions, schools, waiting rooms, offices, clinics and educational activities.

The measurement source must always remain clear: the values come from a Technology Club of Thrace Environmental Station, not from the display application itself.

2. Core Features

What does this specific display provide?

Responsive Dashboard

Measurement display for desktop computers, tablets and mobile devices.

TV Mode

A large-screen version for televisions, using one measurement card at a time.

Bilingual Interface

Greek by default and English through separate translation files.

Date and Time

A card displays the current time and date using the Europe/Athens timezone.

Measurement Descriptions

Each value includes a short explanation of its unit and environmental context.

Visible Data Source

The display keeps the logo, source label and station link visible.

3. Architecture and Installation

How does the application work?

1

Value Source

The application retrieves values from a ThingsBoard endpoint connected to the specific environmental station.

2

Local Configuration

Real connection details are stored in config.php, which is not published to GitHub.

3

Server Cache

Public tokens and telemetry responses are temporarily stored in private/ to reduce upstream load.

Local use

  • Download or clone the code from GitHub.
  • Copy config.example.php to config.php.
  • Fill in the ThingsBoard endpoint details in config.php.
  • Run the application on a local PHP server or in an environment such as XAMPP.

Server use

  • Upload the code to the web server.
  • Create the real config.php only on the server.
  • The private/ folder must be writable by PHP.
  • Cache/token files and real connection details are not published.

4. API Use for Microcontrollers

How are the values displayed on ESP32?

Besides visual display, the application provides small API endpoints for educational electronics projects, ESP32 boards, microcontrollers or simple clients that need selected values.

The API exposes only whitelisted keys and uses the same server-side cache, avoiding unnecessary load on the original measurement source.

All Values

GET /api/v1/latest.php

Compact JSON

GET /api/v1/latest.php?flat=1

Single Value

GET /api/v1/value.php?key=temperature

Plain Text

GET /api/v1/value.php?key=temperature&format=text

Allowed public keys: temperature, humidity, pressure, co, co2, pm25, pm10.

Example ESP32 sketch with an OLED display

The following example reads one value from the lightweight API and displays it on a small OLED screen. It is an educational sample and can be adapted for different displays or libraries. 

#include <WiFi.h>
#include <HTTPClient.h>
#include <Wire.h>
#include <Adafruit_GFX.h>
#include <Adafruit_SSD1306.h>

#define SCREEN_WIDTH 128
#define SCREEN_HEIGHT 64

const char* ssid = "YOUR_WIFI_NAME";
const char* password = "YOUR_WIFI_PASSWORD";
const char* valueUrl = "https://labschool.gr/iot-steth-axd/api/v1/value.php?key=temperature&format=text";

Adafruit_SSD1306 display(SCREEN_WIDTH, SCREEN_HEIGHT, &Wire, -1);

void setup() {
  Serial.begin(115200);
  WiFi.begin(ssid, password);

  display.begin(SSD1306_SWITCHCAPVCC, 0x3C);
  display.clearDisplay();
  display.setTextColor(SSD1306_WHITE);
  display.setTextSize(1);
  display.setCursor(0, 0);
  display.println("Connecting WiFi...");
  display.display();

  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
  }
}

void loop() {
  if (WiFi.status() == WL_CONNECTED) {
    HTTPClient http;
    http.begin(valueUrl);
    int status = http.GET();

    if (status == 200) {
      String temperature = http.getString();

      display.clearDisplay();
      display.setTextSize(1);
      display.setCursor(0, 0);
      display.println("Alexandroupoli");
      display.println("Temperature");
      display.setTextSize(2);
      display.setCursor(0, 28);
      display.print(temperature);
      display.print(" C");
      display.display();
    }

    http.end();
  }

  delay(60000);
}

5. License and Operating Terms

How should the source be credited?

The application is published for educational, informational and community use. Any public display of the values must clearly preserve the original measurement source.

The logo, source label and station link must not be removed, and the values must not be misleadingly presented as data from another source.
